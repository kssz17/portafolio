Principios con los que escribo código

No son trends — son la base con la que construyo software pensado para durar.

01
Código limpio

Nombres claros, funciones cortas y una sola responsabilidad por archivo. Si otra persona no lo lee fácil, lo reescribo.

02
Accesibilidad

HTML semántico, contraste, foco visible y navegación por teclado. La accesibilidad no es opcional, es parte del producto.

03
Testing

Tests donde aporten — pruebas unitarias para lógica crítica, pruebas manuales para flujos de usuario. Confianza al desplegar.

04
Documentación

READMEs útiles, comentarios donde el "porqué" no es obvio, y commits con contexto. El equipo siguiente lo agradecerá.

05
Performance

Lazy loading, optimización de imágenes y queries pensadas. Que el usuario no espere lo que no debe.

06
i18n y escalabilidad

Componentes pensados para crecer y para hablar varios idiomas. El producto de hoy puede ser global mañana.

Mi enfoque

El código que escribo
habla por mí.

Prefiero una función que se entienda al primer intento sobre una "ingeniosa" que requiera explicación. Un buen componente debería decir lo que hace antes de leer su implementación.

  • Componentes pequeños y reutilizables
  • Estado predecible, side effects controlados
  • Tipado fuerte con TypeScript
  • Reviews abiertos al feedback
UserProfile.vue 
// Componente legible > componente clever
const getDisplayName = (user: User): string => {
  if (!user.name?.trim()) return 'Usuario sin nombre';
  return user.name.trim();
};

const canEditProfile = (
  user: User,
  currentUser: User
): boolean => {
  return user.id === currentUser.id
      || currentUser.role === 'admin';
};
